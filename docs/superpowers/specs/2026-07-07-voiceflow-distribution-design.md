# VoiceFlow «Дистрибутив» (спека B) — дизайн-спецификация

Дата: 2026-07-07
Статус: утверждена пользователем
Контекст: упаковка реализованного V2 UX (см. `2026-07-06-voiceflow-v2-ux-design.md`)
для обычных пользователей Windows: установщик, мастер первого запуска,
публичный GitHub-репозиторий с автосборкой релизов.

## Цель

Человек без Python и технических навыков скачивает один exe со страницы GitHub
Releases, ставит его «Далее-Далее», проходит мастер первого запуска и диктует.

## Ключевые решения (зафиксированы пользователем)

- **Лёгкий установщик + докачка**: Setup.exe ~100–150 МБ (приложение +
  Python-рантайм, БЕЗ nvidia-пакетов и моделей); CUDA-библиотеки докачиваются
  только владельцам NVIDIA, модель — по выбору в мастере.
- **Публичный репозиторий + GitHub Actions**: автосборка установщика на тег
  `v*`, автосоздание Release.
- **Без проверки обновлений** в приложении.
- **Стек упаковки**: PyInstaller (onedir, windowed) + Inno Setup.

## Путь пользователя

1. GitHub Releases → скачать `VoiceFlow-Setup-vX.Y.exe`.
2. SmartScreen покажет «Windows защитил ваш компьютер» (подписи нет) →
   в README/релизе скриншот-инструкция «Подробнее → Выполнить в любом случае».
3. Установщик Inno Setup: путь (по умолчанию `%LOCALAPPDATA%\VoiceFlow`,
   БЕЗ прав администратора), галочки «Ярлык на рабочий стол» и «Запускать
   с Windows», по завершении — «Запустить VoiceFlow».
4. Первый запуск → мастер (см. ниже) → приложение в трее.

## Компоненты

### 1. Каталоги данных (`voiceflow/paths.py` — новый)

- `is_frozen() -> bool` — `getattr(sys, "frozen", False)`.
- Dev-режим (venv): всё как сейчас — config/лог в корне репозитория.
- Frozen-режим: данные в `%APPDATA%\VoiceFlow\` (`config.json`,
  `voiceflow.log`, `cuda\` с докачанными DLL); модели — стандартный кэш
  HuggingFace (`%USERPROFILE%\.cache\huggingface`).
- Все обращения к путям в `app.py`/`transcriber.py` идут через `paths.py`.

### 2. Мастер первого запуска (`voiceflow/wizard.py` — новый, tkinter)

Запускается из `app.main()` ДО инициализации остального, если config.json
отсутствует. Одно окно, шаги «Назад/Далее», любой шаг можно пропустить
(дефолты разумные). Шаги:

1. **Приветствие**: что делает приложение, картинка пилюли из assets.
2. **Железо**: `nvidia-smi --query-gpu=name,memory.total --format=csv,noheader`
   через subprocess (нет в PATH / ошибка → «NVIDIA не найдена»).
   NVIDIA найдена → рекомендация large-v3 (радиокнопки трёх пресетов качества
   из настроек, предвыбран large-v3) + пункт «докачать CUDA-библиотеки (~600 МБ)»;
   не найдена → предвыбран base, CUDA-шаг пропускается, device=cpu.
3. **Скачивание** (прогресс-бар + «Отмена»):
   - CUDA: скачать win_amd64-wheels `nvidia-cublas-cu12` и `nvidia-cudnn-cu12`
     с PyPI (JSON API pypi.org → URL последнего win_amd64-wheel), wheel = zip →
     распаковать `*/bin/*.dll` в `%APPDATA%\VoiceFlow\cuda\`; прогресс по байтам.
   - Модель: `huggingface_hub.snapshot_download` для выбранного пресета;
     прогресс — колбэками tqdm-класса (или по росту размера каталога кэша,
     если колбэки недоступны).
   - Ошибка сети → сообщение + «Повторить» / «Пропустить» (пропуск модели =
     скачается при первом распознавании, о чём предупреждаем).
4. **Микрофон**: combobox устройств (как в настройках) + живой индикатор
   уровня: sounddevice-стрим выбранного устройства, RMS → ttk.Progressbar,
   подпись «Скажите что-нибудь — полоска должна дёргаться».
5. **Клавиша**: текущий дефолт `ctrl_left` + кнопка «Изменить» (готовый
   механизм захвата HotkeyListener.start_capture; хук на время мастера
   запускается в отдельном потоке и останавливается по завершении).
6. **Готово**: `save_config`, экран-памятка: «Сейчас загрузится модель
   (уведомление „Готов" в трее), затем зажмите <клавиша> и говорите —
   текст появится там, где стоит курсор». Пробная диктовка происходит уже
   после мастера штатным путём (модель и хук инициализируются после
   закрытия окна).

Мастер закрыт крестиком до конца → config.json сохраняется с дефолтами
(base/cpu если нет NVIDIA), приложение продолжает запуск.

### 3. CUDA-библиотеки в runtime (`voiceflow/transcriber.py` — правка)

`_add_cuda_dll_dirs()` дополнительно ищет `%APPDATA%\VoiceFlow\cuda\`
(через paths.py) и добавляет его в `os.add_dll_directory` + PATH тем же
способом, что и site-packages/nvidia/*/bin (dev-режим не меняется).

### 4. Сборка (`packaging/` — новое)

- `packaging/voiceflow.spec` — PyInstaller: onedir, windowed (без консоли),
  name=VoiceFlow, icon из `packaging/voiceflow.ico` (сгенерировать один раз
  из `voiceflow/assets/icon.png` скриптом `packaging/make_ico.py`),
  datas: `voiceflow/assets/*`; excludes: `nvidia*`; hiddenimports по факту
  (pystray.backend, PIL, sounddevice callback и т.п. — уточняется при
  реализации по ошибкам сборки).
- `packaging/installer.iss` — Inno Setup: PrivilegesRequired=lowest,
  DefaultDirName={localappdata}\VoiceFlow, Tasks: desktopicon,
  autostart (пишет HKCU Run — то же значение `VoiceFlow`, что и настройки),
  Run: «Запустить VoiceFlow» после установки, деинсталлятор удаляет
  и `%APPDATA%\VoiceFlow` по подтверждению.
- `packaging/build.bat` — локальная сборка: pyinstaller → iscc → exe в
  `packaging/dist/`.

### 5. CI/CD (`.github/workflows/`)

- `ci.yml`: на push/PR — windows-latest, установка requirements-dev, pytest.
- `release.yml`: на тег `v*` — pytest → PyInstaller → Inno Setup
  (choco install innosetup) → `VoiceFlow-Setup-<tag>.exe` →
  softprops/action-gh-release с автогенерируемым чейнджлогом.

### 6. Публикация

- Создать публичный репозиторий GitHub `voiceflow` (gh CLI), запушить main.
- README: новый раздел «Установка (для пользователей)» — скачивание из
  Releases, SmartScreen-инструкция, системные требования (Windows 10/11
  x64; NVIDIA GTX 10xx+ для максимального качества, иначе CPU);
  существующие dev-инструкции остаются ниже в разделе «Разработка».
- `.superpowers/` уже в локальном игноре; проверить, что в публичный репо
  не попадает мусор (voiceflow.log, config.json — уже в .gitignore).

## Обработка ошибок

- Любая ошибка мастера не должна ронять приложение: шаг помечается
  пропущенным, приложение стартует с дефолтами; всё в voiceflow.log.
- Обрыв скачивания CUDA/модели → «Повторить»/«Пропустить»; частично
  скачанные wheel-файлы удаляются, распаковка — только после полной загрузки.
- nvidia-smi отсутствует при наличии NVIDIA (экзотика) → пользователь может
  вручную выбрать large-v3 + CUDA-докачку (радиокнопки не блокируются).
- В frozen-режиме фатальный старт по-прежнему пишет «Фатальная ошибка» в лог
  в %APPDATA%\VoiceFlow.

## Тестирование

- Юнит (без сети/железа, с моками): paths.py (frozen/dev), парсинг вывода
  nvidia-smi, выбор рекомендации по железу, URL-выбор wheel из PyPI JSON,
  распаковка DLL из wheel-zip (синтетический zip), «мастер нужен?»
  (нет config.json).
- Ручной чек-лист: локальная сборка build.bat → установка на этой машине →
  мастер целиком (с реальной докачкой CUDA + base-модели) → диктовка →
  деинсталляция; прогон установщика на ПК без NVIDIA (по возможности);
  тег v0.9 → проверка Actions-сборки и Release.
