# VoiceFlow V2 UX — дизайн-спецификация

Дата: 2026-07-06
Статус: утверждена пользователем
Контекст: расширение работающего v1 (см. `2026-07-06-voiceflow-design.md`).
Вторая часть v2 — упаковка/дистрибутив — будет отдельной спекой (exe, установщик,
мастер первого запуска, GitHub Releases) и пишется после реализации этой.

## Цель

Сделать VoiceFlow пригодным для повседневного использования обычным пользователем:
настраиваемые hotkey-комбинации, окно настроек вместо правки config.json,
красивый оверлей на изображениях, звуковые сигналы записи, автозапуск.

## Требования

### 1. Hotkey-комбинации

- Конфиг: `"hotkey"` — список имён клавиш, например `["ctrl_left", "alt_left", "x"]`;
  одиночная клавиша — список из одного элемента.
- Миграция v1: строки `"left_ctrl"` / `"right_ctrl"` (и любое строковое значение)
  автоматически конвертируются в список при загрузке конфига.
- Таблица имя↔VK в `hotkey.py`: буквы A–Z, цифры 0–9, F1–F24, модификаторы
  с различием сторон (`ctrl_left`, `ctrl_right`, `alt_left`, `alt_right`,
  `shift_left`, `shift_right`, `win_left`, `win_right`), `space`, `capslock`,
  `tab`, `backquote`, стрелки, `insert`/`delete`/`home`/`end`/`pageup`/`pagedown`.
- Push-to-talk для комбинации: **все клавиши комбинации зажаты → on_press;
  любая из них отпущена → on_release**; повторный on_press возможен только после
  того, как каждая клавиша комбинации была отпущена хотя бы один раз с момента
  последнего срабатывания (одновременного полного отпускания не требуется —
  «перекатное» отпускание тоже взводит комбинацию заново); дожатие одной
  клавиши при остальных удерживаемых повторный on_press не вызывает.
- Хук игнорирует инжектированные события (`KBDLLHOOKSTRUCT.flags & LLKHF_INJECTED`),
  иначе собственная эмуляция Ctrl+V дёргала бы hotkey, содержащий Ctrl.
- Режим захвата для настроек: `HotkeyListener.start_capture(callback)` — хук
  перестаёт вызывать on_press/on_release и копит «пиковое множество» одновременно
  зажатых клавиш; когда все отпущены — `callback(соответствующий список имён)`,
  режим захвата выключается. Клавиши без имени в таблице игнорируются при захвате.

### 2. Окно настроек

- Пункт «Настройки…» в меню трея (первым пунктом). Окно — `tk.Toplevel` от
  существующего tk-root оверлея; создание запрашивается через ту же
  потокобезопасную очередь (пункт меню pystray работает из чужого потока).
  Повторное открытие поднимает существующее окно, а не плодит копии.
- Контролы (ttk):
  - **Hotkey**: метка с текущей комбинацией («Ctrl слева + Alt слева + X»)
    + кнопка «Изменить» → текст меняется на «Нажмите комбинацию…», захват
    через start_capture, по завершении — новая комбинация в метке.
  - **Микрофон**: ttk.Combobox, элементы — устройства ввода из
    `sounddevice.query_devices()` (уникальные по имени, hostapi WASAPI/MME —
    показывать по одному разу) + первый пункт «Системный по умолчанию» (None).
  - **Язык**: ttk.Combobox: «Авто» (auto) / «Русский» (ru) / «English» (en).
  - **Качество**: ttk.Combobox из пресетов:
    «Максимальное — large-v3 (нужна NVIDIA, ~3 ГБ)» → model `large-v3`,
    «Среднее — small (~500 МБ)» → model `small`,
    «Быстрое — base (~150 МБ)» → model `base`.
  - **Звуки записи**: ttk.Checkbutton → конфиг `sounds`.
  - **Автозапуск с Windows**: ttk.Checkbutton → значение `VoiceFlow`
    в `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` (модуль `winreg`).
    Команда автозапуска: текущий способ запуска приложения (в v2 — полный путь
    к `voiceflow.bat`; модуль автозапуска должен принимать команду параметром,
    чтобы спека B подставила путь к exe).
  - Кнопки **Сохранить** / **Отмена**.
- Применение изменений после «Сохранить»:
  - hotkey, звуки, автозапуск — мгновенно;
  - микрофон — мгновенно (`recorder.device`), действует со следующей записи;
  - язык и модель — фоновая перезагрузка транскрайбера: уведомление в трее
    «Загружаю модель…», на время загрузки диктовка выключена (on_press
    игнорируется), по завершении — уведомление «Готов». Ошибка загрузки новой
    модели → уведомление, откат на прежнюю модель (старый транскрайбер
    сохраняется до успешной загрузки нового).
- `config.json` перезаписывается целиком со всеми ключами.

### 3. Оверлей на изображениях

- `voiceflow/assets/recording.png` и `voiceflow/assets/transcribing.png` —
  PNG RGBA 400×128 (генерирует пользователь через Gemini; встроенные по
  умолчанию). Требование к картинкам: карточка целиком нарисована внутри
  изображения, без полупрозрачных краёв/теней (прозрачность реализуется
  цветовым ключом).
- Показ: Pillow загружает PNG, масштабирует под DPI окна
  (`GetDpiForWindow`: 96 DPI → 200×64, 192 DPI → 400×128, промежуточные —
  пропорционально), компонует alpha на ключевой цвет `#FF00FE`;
  окно — `overrideredirect` + `-transparentcolor #FF00FE`, WS_EX_NOACTIVATE
  сохраняется как в v1.
- Если файлы отсутствуют или не читаются — откат на текстовый вид v1
  (он же используется в автотестах, чтобы не зависеть от бинарных ассетов).

### 4. Звуки записи

- `voiceflow/assets/start.wav`, `voiceflow/assets/stop.wav` — короткие (<0.2 с)
  тихие сигналы (сгенерировать синтетически при реализации: два синус-тика
  разной высоты, ~800/500 Гц).
- Воспроизведение stdlib-модулем `winsound`, флаги `SND_FILENAME | SND_ASYNC`,
  в момент фактического старта записи и постановки в очередь распознавания.
- Конфиг `sounds: true|false`; отсутствие wav-файлов не ломает работу (тихо
  пропускается с log.warning один раз).

### 5. Конфиг

Новые/изменённые ключи (`config.DEFAULTS`):

| Ключ | Было (v1) | Стало (v2) |
|---|---|---|
| hotkey | `"right_ctrl"` (строка) | `["ctrl_left"]` (список имён) |
| sounds | — | `true` |
| model | `"large-v3"` | без изменений; в GUI управляется пресетом качества |

Миграция: строковый `hotkey` → список (`"left_ctrl"`→`["ctrl_left"]`,
`"right_ctrl"`→`["ctrl_right"]`, прочие строки → `["ctrl_right"]` c warning).
`save_config(path, cfg)` — новая функция записи (JSON, utf-8, indent=2).

## Архитектура изменений

| Модуль | Изменение |
|---|---|
| `hotkey.py` | таблица VK-имён; логика комбинаций (множество зажатых); игнор injected; режим захвата |
| `config.py` | новые ключи, миграция hotkey, `save_config` |
| `overlay.py` | картинки + DPI + transparentcolor, откат на текст; канал «показать настройки» в той же очереди |
| `settings.py` | **новый**: окно настроек (Toplevel), чтение/запись конфига, применение изменений через колбэки |
| `autostart.py` | **новый**: `is_enabled() -> bool`, `enable(command: str)`, `disable()` через winreg |
| `sounds.py` | **новый**: `play_start()`, `play_stop()`, `set_enabled(bool)` |
| `tray.py` | пункт меню «Настройки…» |
| `controller.py` | вызовы sounds; метод `set_recorder_device`; блокировка диктовки на время перезагрузки модели |
| `app.py` | wiring: settings-колбэки, фоновая перезагрузка транскрайбера |
| `transcriber.py` | без изменений (пересоздаётся целиком) |

Поточная модель не меняется: tk — главный поток (настройки — Toplevel там же),
хук — свой поток, распознавание — worker, pystray — detached.

## Обработка ошибок

- Ошибка загрузки новой модели → уведомление + откат на старую (см. выше).
- Реестр недоступен (правы урезаны) → чекбокс автозапуска показывает ошибку
  уведомлением, конфиг не ломается.
- Захват hotkey отменяется клавишей Esc (Esc одиночный не может быть hotkey).
- Битые/отсутствующие PNG или WAV → откат на текст / тишину, warning в лог.

## Тестирование

- Юнит (pytest, без железа): комбо-логика (полное зажатие, частичное отпускание,
  дребезг, injected-фильтр), захват пикового множества + Esc-отмена, миграция
  конфига, `save_config` round-trip, маппинг пресетов качества, autostart
  (с моком winreg), sounds (с моком winsound).
- Ручной чек-лист: смена hotkey на Ctrl+Alt+X и диктовка; выбор микрофона;
  смена качества с перезагрузкой модели на лету; звуки вкл/выкл; автозапуск
  (проверить значение в реестре); оверлей-картинки на 100% и 150% DPI;
  Ctrl+V-вставка не дёргает hotkey с Ctrl в составе.
