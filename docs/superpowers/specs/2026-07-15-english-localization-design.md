# Английская версия Molvi: локализация приложения и сайта

Дата: 2026-07-15. Статус: утверждено.

## Цель

Сделать Molvi доступным англоязычной аудитории: интерфейс приложения на
английском (с переключением RU/EN) и англоязычная версия сайта molvi.tech.

## Ключевые решения

- **Один экземпляр приложения** с настройкой языка интерфейса, а не отдельная
  EN-сборка. При первом запуске язык берётся из системной локали.
- **Язык распознавания** уже настраиваемый (`auto|ru|en` в конфиге) — ничего
  не меняем, дефолт `auto` (Whisper сам определяет язык речи).
- **Механизм i18n** — свой модуль-словарь `molvi/i18n.py` (без gettext и
  внешних файлов): ноль зависимостей, не трогает PyInstaller-сборки,
  тестируется на полноту ключей.
- **Сайт** — английская версия по пути `molvi.tech/en/`, русская остаётся на
  корне. Перекрёстный hreflang, переключатель RU/EN в шапке.

## Часть 1: приложение

### Модуль `molvi/i18n.py`

Единый источник строк по образцу `theme.py`:

- Словари `RU` и `EN` с одинаковым набором ключей
  (`"settings.hotkey_label"`, `"tray.quit"`, …).
- `tr(key, **fmt)` — возврат строки текущего языка с подстановкой через
  `str.format` («Готов. Зажмите {hotkey} и говорите»). При отсутствии ключа —
  fallback на сам ключ (не падать).
- `set_language(lang)` — переключение на лету.
- Определение системной локали для режима `auto`: модуль `locale`,
  `ru*` → ru, всё остальное → en.

### Конфиг

Новый ключ `ui_language: "auto" | "ru" | "en"`, дефолт `"auto"`.
Миграция не нужна: отсутствующий ключ падает в дефолт.

### Замена строк (~150–180 видимых пользователю)

Файлы: `wizard.py`, `settings.py`, `app.py`, `tray.py`, `controller.py`,
`overlay.py`, `transcriber.py`, `hotkey.py`, `platform/darwin/hotkey.py`.

Особые точки:

- Таблицы подписей клавиш — `VK_NAMES` (win32) и `TABLE` (darwin) получают
  ru/en-варианты: «Пробел»/«Space», «Ctrl слева»/«Left Ctrl»,
  «⌘ Cmd справа»/«Right ⌘ Cmd».
- Единицы «ГБ/МБ» → «GB/MB» (метки VRAM и размеров загрузок в мастере и
  настройках).
- Логи (`log.*`) остаются русскими — их читает разработчик, не пользователь.

### UI выбора языка

- **Настройки**: комбобокс «Язык интерфейса» (Авто / Русский / English) рядом
  с существующим «Язык» (распознавания). Смена применяется сразу: меню трея
  пересоздаётся, оверлей читает строки при показе; окно настроек
  перерисовывается при следующем открытии.
- **Мастер первого запуска**: выбор языка — первый экран (RU/EN), чтобы
  англоязычный пользователь сразу видел мастер на своём языке. Выбор
  записывается в `ui_language`.

### Тесты

- RU и EN содержат одинаковые множества ключей.
- В EN-словаре нет кириллицы.
- `tr()` корректно подставляет параметры; при неизвестном ключе возвращает
  ключ.
- Определение локали: `ru_RU` → ru, `en_US`/прочие → en.

## Часть 2: сайт molvi.tech/en/

- Переводы 5 страниц в `docs/site/en/` с той же структурой:
  `en/index.html`, `en/install.html`, `en/install-mac.html`,
  `en/geek/index.html`, `en/geek/mac/index.html`.
- `lang="en"`, переведённые title/description/OG, `og:locale`.
- На всех страницах (RU и EN) — перекрёстные
  `<link rel="alternate" hreflang="ru|en|x-default">` (x-default → русская
  версия) и видимый переключатель RU/EN в шапке.
- `sitemap.xml` — добавить 5 английских URL с языковыми альтернативами.
- Деплой не меняется (rsync заберёт `en/`); в воркфлоу добавить smoke-check
  на `https://molvi.tech/en/`.
- **Скриншоты**: на EN-страницах первое время остаются текущие (русский
  мастер). После релиза локализованного приложения — переснять английские и
  заменить. Осознанный компромисс, чтобы не блокировать сайт релизом.

## Порядок работ

1. i18n-модуль + перевод приложения + тесты (даст материал для английских
   скриншотов).
2. Английский сайт + hreflang + sitemap + smoke-check.
3. После релиза — английские скриншоты на EN-страницы.

## Вне рамок

- Перевод логов и комментариев в коде.
- Другие языки кроме RU/EN (структура i18n.py их не исключает).
- Отдельные EN-сборки/установщики.
- Немедленная пересъёмка скриншотов для сайта.
