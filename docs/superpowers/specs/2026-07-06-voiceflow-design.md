# VoiceFlow — дизайн-спецификация

Дата: 2026-07-06
Статус: утверждена пользователем

## Цель

Локальный аналог Wispr Flow для Windows: по зажатию правого Ctrl записывает голос,
распознаёт его локально через Whisper и вставляет текст в активное приложение
(в том числе в терминальную сессию с Claude Code). Без подписок и облачных сервисов.

## Требования

- **Движок STT:** faster-whisper (CTranslate2, CUDA) с моделью large-v3 на RTX 4080,
  квантизация int8_float16 (~3 ГБ VRAM). Автоматический откат на CPU при отсутствии CUDA.
- **Языки:** русский + английский вперемешку, автоопределение языка на каждую фразу.
- **Hotkey:** push-to-talk на правом Ctrl — зажал → запись, отпустил → распознавание и вставка.
- **Постобработка LLM:** отсутствует в v1 (Whisper сам расставляет пунктуацию).
- **UI:** иконка в трее (статус, пауза, выход) + мини-оверлей поверх окон
  («● Запись…» / «⏳ Распознаю…»).
- **Платформа:** Windows 10, Python 3.12 в venv (3.13 — если бинарные колёса
  ctranslate2 доступны и работают).

## Архитектура

Один фоновый Python-процесс без консоли (`pythonw`), живёт в трее.

### Поток данных

1. Низкоуровневый хук клавиатуры (WH_KEYBOARD_LL) ловит нажатие правого Ctrl →
   старт записи с микрофона (16 кГц, моно) в numpy-буфер в памяти → оверлей «● Запись…».
2. Отпускание правого Ctrl → стоп записи → оверлей «⏳ Распознаю…» →
   аудио передаётся в faster-whisper (large-v3, автоопределение языка).
3. Текст вставляется в активное окно: сохранить текущий буфер обмена →
   положить текст в буфер → эмулировать Ctrl+V → после короткой паузы восстановить буфер.
   Оверлей скрывается.

### Ключевые свойства

- Модель загружается один раз при старте и остаётся в VRAM — задержка от отпускания
  клавиши до вставки текста ~1–2 сек.
- Записи короче 0.3 сек игнорируются (случайные нажатия правого Ctrl).
- Распознавание выполняется в отдельном потоке; хук клавиатуры и трей не блокируются.
- Пока идёт распознавание предыдущей фразы, новая запись допустима — фразы обрабатываются
  последовательно через очередь.
- Конфигурация в `config.json` рядом с приложением: клавиша, модель, язык (auto/ru/en),
  устройство записи, размер квантизации.

## Компоненты

| Модуль | Ответственность | Зависимости |
|---|---|---|
| `hotkey.py` | Хук WinAPI: события «правый Ctrl нажат/отпущен», колбэки press/release | ctypes/WinAPI |
| `recorder.py` | Запись с микрофона в numpy-буфер: `start()` / `stop() → np.ndarray` | sounddevice, numpy |
| `transcriber.py` | Загрузка модели, `transcribe(audio) → str` | faster-whisper |
| `typer.py` | Вставка текста через буфер обмена + Ctrl+V с восстановлением буфера; запасной путь — посимвольный SendInput | pywin32/ctypes |
| `overlay.py` | Tkinter-окно без рамки поверх всех окон, состояния idle/recording/transcribing | tkinter |
| `tray.py` | Иконка в трее: статус, пауза/возобновление, выход | pystray, Pillow |
| `app.py` | Точка входа: конфиг, связывание модулей, очередь обработки, логирование | — |

Каждый модуль тестируем независимо: у каждого одна ответственность и узкий интерфейс.

## Обработка ошибок

- Микрофон недоступен → уведомление + запись в лог; приложение продолжает работать
  (повторная попытка при следующем нажатии).
- CUDA недоступна → автоматический откат на CPU с уведомлением о сниженной скорости.
- Whisper вернул пустой текст (тишина) → ничего не вставляем, оверлей скрывается.
- Ошибка вставки → текст остаётся в буфере обмена, уведомление «текст в буфере,
  вставьте вручную (Ctrl+V)» — диктовка не теряется.
- Все ошибки пишутся в `voiceflow.log` (ротация, максимум ~1 МБ).

## Тестирование

- Юнит-тесты (pytest) на логику без железа: чтение/дефолты конфига, фильтр коротких
  записей, последовательность сохранения/восстановления буфера обмена (с моками).
- Ручная end-to-end проверка: диктовка в Notepad, в терминал с Claude Code, в браузер;
  смешанная русско-английская фраза; случайное короткое нажатие Ctrl; диктовка при
  занятом распознавании.

## Запуск и автозагрузка

- `venv` в каталоге проекта, `requirements.txt`.
- `voiceflow.bat` для ручного запуска; ярлык на `pythonw.exe app.py` в папке
  автозагрузки для старта с Windows.

## Вне рамок v1 (осознанно)

- LLM-постобработка текста (фильтр «эмм», причёсывание формулировок).
- Настройки через GUI (только config.json).
- Установщик/exe-дистрибутив.
- Режим toggle и настраиваемые комбинации клавиш (только правый Ctrl).
