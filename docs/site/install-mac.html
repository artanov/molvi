Установка Molvi на Mac

Пять шагов, 5–10 минут вместе с загрузкой модели. Нужен Mac на Apple Silicon (M1 и новее), macOS 13.5+.

Скачайте образ

Нажмите кнопку — откроется страница последнего релиза на GitHub. Скачайте файл Molvi-…dmg (~150 МБ) из блока «Assets».

Скачать последний релиз

Почему GitHub

Molvi — программа с открытым кодом, и GitHub — её официальный и единственный источник. Образы из других мест могут быть подделкой.

Установите и пройдите Gatekeeper

Откройте DMG и перетащите Molvi на ярлык Applications. Затем — важный момент первого запуска:

  • откройте папку «Программы» и нажмите на Molvi правой кнопкой;
  • выберите «Открыть», и в появившемся окне — снова «Открыть».

Не удаётся подтвердить разработчика «Molvi»

macOS не может проверить приложение на наличие вредоносного ПО. Открывайте, только если уверены в источнике.

Открыть Отмена

Так выглядит предупреждение Gatekeeper (иллюстрация). Через правый клик появляется кнопка «Открыть» — при обычном двойном клике её нет.

Почему появляется это окно

Apple доверяет приложениям с нотаризацией (платная подписка разработчика, $99 в год). У бесплатного Molvi её нет, поэтому Gatekeeper перестраховывается. Код Molvi открыт — любой может проверить, что внутри. Альтернатива для любителей терминала: xattr -cr /Applications/Molvi.app

Пройдите мастер первого запуска

Molvi сам предложит всё настроить — несколько коротких экранов.

  • РекомендуемМаксимальное — large-v3-turbo — качество уровня large-v3, но быстрое: Apple Silicon распознаёт через Metal (движок mlx-whisper). Загрузка ~1.6 ГБ, хватает даже M1 с 8 ГБ памяти.
  • Если совсем мало местаsmall / base — меньше и быстрее скачиваются, качество ниже.
  • на шаге «Загрузка компонентов» нажмите «Начать загрузку» и дождитесь «Готово!»;
  • на шаге «Микрофон» скажите что-нибудь — полоска должна дёргаться (macOS спросит доступ к микрофону — разрешите);
  • клавиша диктовки по умолчанию — правый ⌘; менять не обязательно, в любой момент можно в Настройках.

Модель качается с Hugging Face (mlx-community) — открытая конвертация Whisper от OpenAI. Сохраняется в ~/.cache/huggingface.

Выдайте два разрешения macOS

Molvi слушает клавишу диктовки и вставляет текст в активное окно — macOS требует явно разрешить и то, и другое. Мастер покажет отдельный шаг с кнопками «Выдать…», которые открывают нужные панели System Settings:

  • ОбязательноМониторинг ввода (Input Monitoring) — без него Molvi не видит клавишу диктовки.
  • ОбязательноУниверсальный доступ (Accessibility) — без него Molvi не может вставить распознанный текст.
  • включите галочку Molvi в обеих панелях;
  • после выдачи перезапустите Molvi (значок в строке меню → «Выход», затем открыть заново) — macOS применяет разрешения при старте.
Почему это безопасно

Molvi использует эти права только для клавиши диктовки и вставки текста — весь код открыт, распознавание не покидает ваш компьютер: ни звук, ни текст никуда не отправляются.

Продиктуйте первую фразу

  • дождитесь уведомления «Готов» (значок Molvi живёт в строке меню, справа сверху);
  • поставьте курсор в любое текстовое поле — заметки, браузер, терминал;
  • зажмите правый ⌘ — появится пилюля «Запись…»;
  • скажите: «Привет! Это Molvi печатает моим голосом»;
  • отпустите клавишу — через секунду-другую текст появится у курсора.
Если хочется что-то поменять

Значок Molvi в строке меню → «Настройки…»: клавиша, микрофон, язык, качество модели, звуки, автозапуск.

Если что-то пошло не так

Обновил Molvi — перестала работать клавиша или вставка

При обновлении неподписанного приложения macOS сбрасывает выданные разрешения. В System Settings → Privacy & Security выключите-включите галочки Molvi в «Мониторинге ввода» и «Универсальном доступе» (или удалите Molvi из списка и добавьте заново) и перезапустите приложение.

Пришло уведомление «нет разрешения „Универсальный доступ“»

Вставка не сработала, но текст не потерян — он остался в буфере обмена, нажмите ⌘V. Выдайте разрешение (шаг 4) и перезапустите Molvi — дальше текст будет вставляться сам.

Первая диктовка после запуска долгая

Так и должно быть: при первом распознавании Metal компилирует вычислительные ядра (несколько секунд). Уже вторая фраза распознаётся за одну-две секунды.

Текст не появляется

Проверьте микрофон в Настройках (значок в строке меню) и убедитесь, что курсор стоит в текстовом поле. Журнал ошибок: ~/Library/Application Support/Molvi/molvi.log. Не решилось — создайте issue.

У меня Mac на Intel

Пока не поддерживается: распознавание работает через Apple-движок MLX, которому нужен чип M-серии (M1 и новее).
